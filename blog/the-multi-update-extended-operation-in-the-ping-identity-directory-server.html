The Ping Identity Directory Server offers a multi-update extended operation that allows you to bundle multiple write operations (which can optionally be processed as a single atomic unit) into the same request. This offers better performance and lower latency than sending the requests one after another. I wrote about it on my personal blog at https://nawilson.com/2019/02/19/the-multi-update-extended-operation-in-the-ping-identity-directory-server/.

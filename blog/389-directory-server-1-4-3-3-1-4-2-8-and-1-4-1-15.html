The 389 Directory Server project has announced new releases of versions 1.4.3.3, 1.4.2.8, and 1.4.1.15. From the release announcements, it looks like some of the changes in these versions are:

  • Fixed an issue that could cause database and cache corruption if a deadlock was encountered during modify DN processing (all three versions)
  • Fixed a crash resulting from a null pointer dereference that could occur when attempting to update the configuration (version 1.4.3.3)
  • Fixed a crash that could occur during SASL bind processing (version 1.4.1.15)
  • Fixed a memory leak that could occur in evaluating access control rules based on IP addresses (all three versions)
  • Fixed an issue that could cause an error log message during modify DN processing (all three versions)
  • Fixed a number of health checking issues (all three versions)
  • Updated the server to enforce a hard upper limit on file descriptor usage< (all three versions)
  • Fixed an issue that could affect the ability to use the unique attribute plugin (version 1.4.3.3)
  • Added support for only listening for LDAPS connections (version 1.4.3.3)
  • Added the ability to automatically load PEM-formatted certificates into the certificate database during startup (version 1.4.3.3)

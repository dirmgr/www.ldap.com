The 389 Directory Server project has released version 2.4.5. Some of the changes in this release appear to include:

  • Fixed a potential crash that could occur if a paged results search is abandoned
  • Fixed a locking issue that could occur when reverting the cached representation of an entry
  • Fixed cache consistency issues that could arise if the BETXN plugin is enabled and a transaction is aborted
  • Added support for an inChain matching rule
  • Disabled support for transparent huge pages by default, as it can reduce performance when using large caches

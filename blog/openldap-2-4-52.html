The OpenLDAP project has announced the release of OpenLDAP version 2.4.52. From the release announcement, some of the changes in this release appear to be:

  • Fixed potential memory corruption issues resulting from freeing unallocated memory
  • Fixed a replication issue that could crash the server
  • Fixed an issue that could cause index corruption when processing a delete operation
  • Fixed a potential replication deadlock that could arise as a result of network issues
  • Added an option to control how certificate hostname validation can be performed when establishing TLS connections
  • Added support for configuring multiple EECDH curves for use in TLS negotiation
  • Fixed a potential hang when attempting to establish a UDP-based connection to a server that is unavailable

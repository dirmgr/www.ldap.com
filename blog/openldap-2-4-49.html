The OpenLDAP project has announced the release of OpenLDAP version 2.4.49. Although the release announcement only contains very cursory information about each of the changes, it appears to be primarily a bugfix release that includes the following:

  • Addressed some memory and file descriptor management issues
  • Added a database entry count to the monitor for the slapd-mdb backend
  • Addressed issues around cancel and abandon operations
  • Avoided acquiring a global lock when using GnuTLS
  • Fixed an issue in handling the preferredDeliveryMethod syntax
  • Improved error handling during connection setup
  • Fixed an issue involving incorrect handling of attribute descriptions
  • Fixed an issue with database interaction during an index update
  • Addressed a couple of password policy issues

The 389 Directory Server project has announced new releases of versions 1.4.4.3, 1.4.3.9, and 1.4.2.14. From the release announcements, it looks like some of the changes in these versions are:

  • Addressed issues that could prevent the server from starting in a container (all three versions)
  • Addressed memory leaks (all three versions)
  • Abort an operation if a CSN cannot be generated for it (all three versions)
  • Allow the uid attribute in the replication manager entry (all three versions)
  • Updated ds-replcheck to make the online timeout configurable (all three versions)
  • Improved instance name validation for the dscreate tool (all three versions)
  • Fixed issues with the healthcheck tool (all three versions)
  • Added support for rewriting filters that use the Active Directory objectCategory attribute (versions 1.4.4.3 and 1.4.3.9)
  • Added support for rewriting filters that use the Active Directory objectSID attribute (version 1.4.4.3)
  • Fixed an issue that could make it expensive to identify non-leaf entries in a directory with a lot of those entries (version 1.4.4.3)

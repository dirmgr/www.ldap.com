The 389 Directory Server project has announced the release of version 1.4.3.19. From the release announcements, it looks like some of the changes in these versions are:

  • Fixed an issue that could disclose the existence of an entry when trying to bind as that user with no password
  • Fixed a number of memory management issues
  • Fixed an issue that could leave the database in a broken state after a failed re-indexing attempt
  • Fix an issue that could cause the server to choose the wrong operation when replaying replicated changes
  • Fix an issue affecting the server when running on big endian systems
  • Improve entry cache access performance
  • Improved dsconf root DN access control validation

The 389 Directory Server project has announced new releases of versions 1.4.3.5, 1.4.2.11, and 1.4.1.17. From the release announcements, it looks like some of the changes in these versions are:

  • Fixed memory leaks in disk monitoring (all three versions)
  • Fixed memory leaks in indirect CoS (all three versions)
  • Fixed memory leaks in the chaining plugin (version 1.4.3.5)
  • Fixed an issue with the “-n” argument to dbverify (all three versions)
  • Fixed an issue in which wildcards were not handled properly in the rootdn-allow-ip configuration attribute (versions 1.4.1.17 and 1.4.3.5)
  • Fixed an issue in which dscreate could fail in silent mode (version 1.4.3.5)
  • Fixed an issue in which dscreate could fail in silent mode (version 1.4.3.5)
  • Added support for building on FreeBSD (version 1.4.3.5)

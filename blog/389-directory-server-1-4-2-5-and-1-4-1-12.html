The 389 Directory Server project has released versions 1.4.2.5 and 1.4.1.12 (the latter released just after version 1.4.11 to fix one additional issue, so those fixes are rolled up together). Changes included in these releases are:

  • Fixed an issue that could cause database recovery to fail if an empty directory exists in the database directory (1.4.2.5 only)
  • Fixed an issue that could cause the server to report a disorderly shutdown (1.4.2.5 only)
  • Fixed an issue in processing long lines in the DSE.ldif (1.4.1.12 only)
  • Fixed an issue that could cause the server to hang (both versions)
  • Fixed an issue that could cause changelog trimming to crash at shutdown if it had an invalid configuration (both versions)
  • Fixed an issue that prevented dumping the changelog to a file (both versions)
  • Updated docker configuration for Fedora (both versions)
  • Enable argument completion in the CLI (both versions)
  • Added additional health checks to dsconfg (both versions)

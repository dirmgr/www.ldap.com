The 389 Directory Server project has released version 2.0.14. Changes in this release appear to include:

  • Fixed a potential crash as a result of changelog trimming
  • Fixed an issue that could cause index creation to fail
  • Fixed an issue with migration of indexes from OpenLDAP
  • Fixed an issue with the mechanism used to specify the primary replica in a winsync environment
  • Fixed an issue with setting a disk monitoring threshold
  • Fixed an issue in the way the container socket was configured for ldapi

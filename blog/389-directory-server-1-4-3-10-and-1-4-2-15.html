The 389 Directory Server project has announced new releases of versions 1.4.3.10 and 1.4.2.15. From the release announcements, it looks like some of the changes in these versions are:

  • Fixed an issue in which the pwdReset attribute was user-writable (both versions)
  • Fixed an issue in which an entry could be missing a DN (version 1.4.3.10)
  • Fixed an issue with numSubordinates in the cn=monitor entry (both versions)
  • Fixed an issue with dsctl and dsidm error reporting when using JSON (both versions)
  • Set a default minimum number of worker threads (both versions)
  • Improved automatic tuning defaults (both versions)

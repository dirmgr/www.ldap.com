The 389 Directory Server project has announced new releases of versions 1.4.3.2, 1.4.2.7, and 1.4.1.14. From the release announcements, it looks like some of the changes in these versions are:

  • Set the default TLS version to TLSv1.2 (all three versions)
  • Addressed several memory leaks (all three versions)
  • Fixed a memory management issue in rotated log cleanup handling (all three versions)
  • Fixed an issue that allowed the root password to be set to an empty string (all three versions)
  • Include a diagnostic message in the error result when an invalid search filter was provided (all three versions)
  • Fixed an issue in which the server could report a disorderly shutdown (versions 1.4.3.2 and 1.4.2.7)
  • Enabled attribute syntax validation by default (version 1.4.1.14)

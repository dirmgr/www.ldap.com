The 389 Directory Server project has released version 1.4.4.6. Some of the changes in this release include:

  • Fixed an issue in which certain search operations could trigger a denial of service.
  • Fixed an issue in which large imports could fail because of a task timeout.
  • Fixed an issue in which the chaining backend did not properly validate bind parameters.
  • Fixed a replication issue in which an internal structure could be incorrectly initialized.
  • Fixed a replication issue in syncrepl support that could cause the wrong cookie value to be used when a series of changes target the same entry.
  • Fixed an issue in which importing data into the changelog could incorrectly close all changelogs instead of just the one being imported.
  • Fixed a misleading access log message when a connection was closed because of an idle timeout.
  • Fixed an issue that could cause an incorrectly formatted message to be written to the server error log.
  • Fixed an issue that could ds-replcheck to crash when processing glue entries.
  • Fixed an issue in which disallowed repeated character sequences in a password could go undetected.

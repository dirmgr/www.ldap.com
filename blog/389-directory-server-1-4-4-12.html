The 389 Directory Server project has released version 1.4.4.12. Some of the changes in this release include:

  • Fix a memory leak in code for maintaining the server root DSE
  • Fix a use-after-free issue during replication conflict resolution
  • Fix an issue that could cause the server to choose the wrong operation when replaying replicated changes
  • Fix an issue affecting the server when running on big endian systems
  • Fix a regression in the result returned when searching in a nonexistent subtree
  • Improve entry cache access performance
